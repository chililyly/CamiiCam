# CamiiCam Photobooth (React + Supabase)

A photobooth website inspired by your provided design with these sections:

- Hero start section
- Layout chooser
- Capture process with timer + filters
- Live webcam stream capture
- Downloadable photo strip export
- FAQ
- Privacy Policy
- About + Contact form

The frontend is React (Vite) and backend integration is Supabase.

## 1. Install and run

```bash
npm install
npm run dev
```

## 2. Environment variables

Copy `.env.example` to `.env` and set:

```env
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=your-public-anon-key
```

## 3. Supabase setup

Create these resources in your Supabase project.

If you use Supabase CLI migrations, run:

```bash
supabase db push
```

Migration files are in `supabase/migrations`.

### Storage bucket

- Bucket name: `photobooth-uploads`

### Table: `photo_sessions`

```sql
create table if not exists photo_sessions (
	id bigint generated by default as identity primary key,
	session_id uuid not null,
	layout_id text not null,
	filter_id text not null,
	total_shots int not null,
	created_at timestamptz not null default now()
);
```

### Table: `contact_messages`

```sql
create table if not exists contact_messages (
	id bigint generated by default as identity primary key,
	name text not null,
	email text not null,
	message text not null,
	created_at timestamptz not null default now()
);
```

## 4. Notes

- If Supabase env values are missing, the site still runs and keeps sessions locally in memory.
- Uploading photos stores files in `photobooth-uploads/{session_id}/shot-n.jpg`.
- Contact form inserts into `contact_messages` when Supabase is configured.
- The app supports webcam mode and upload mode for capture source.
- Export button downloads the generated strip as a JPEG file.
- The RLS migration enforces admin-only reads for `contact_messages`.
